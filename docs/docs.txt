📄 Technical Documentation: Notification-as-a-Service (NaaS)
1. Architectural OverviewThis system is a multi-tenant Notification-as-a-Service (NaaS) engine built with 
NestJS, Prisma, and PostgreSQL. It allows different software platforms (Tenants) to send notifications to their
 users (Contacts) via customizable layouts (Templates) securely using cryptographically hashed API keys.
 
2. SOLID Design Principles AppliedSingle Responsibility Principle (SRP): * NotificationController: Handles HTTP 
requests and formatting only.NotificationService: Orchestrates multi-tenant business logic.TemplateCompilerService: Is 
isolated to handle text interpolation (Handlebars). Swapping engines will not break database or routing logic.PrismaService: 
Manages database connections exclusively.
